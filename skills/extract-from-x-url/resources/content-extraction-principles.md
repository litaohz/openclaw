# 内容提取原则与最佳实践

本文档总结了从 X/Twitter 链接提取 AI 动态新闻时的核心原则，旨在防止"过度依赖外部链接而忽视推文本身"的错误。

## 核心原则：推文即新闻 (Tweet is the News)

在处理一条 X 链接及其关联的外部链接（如 Blog、Paper、HuggingFace）时，必须遵循以下优先级：

### 1. 第一现场优先 (News Hierarchy)

*   **最高优先级：推文正文 & 配图**
    *   推文正文通常包含了**"刚刚发生了什么"**（What happened just now），例如："刚刚登顶第一"、"视频时长提升到10s"、"发布了"、"更新了"。
    *   **这是新闻点**。
*   **辅助优先级：外部链接内容**
    *   外部链接（论文、博客）通常解释**"它是什么"**（What is it）、技术原理、架构细节。
    *   **这是背景知识**。

**🚫 常见错误**：看到推文里有个论文链接，就直接去读论文，写了一堆复杂的模型架构，却漏掉了推文里写的"刚刚开源"或"性能提升了50%"这样的核心动态。

### 2. 视觉信息挖掘 (Visual Information Mining)

很多 AI 产品的核心卖点（Version号、具体指令、Demo效果）并不在文字里，而是在**配图**或**视频封面**上。

*   **必须检查**：推文配图中的文字。
*   **示例**：
    *   推文只说"New update"，但配图里写了 "GPT-5.2 integrated"。
    *   推文只说"Better video"，但配图右下角显示时长 "00:10"。

### 3. "定义"与"新闻"的分离

在撰写"核心内容"时，应清晰区分这两个维度：
*   **新闻点 (News)**：来自推文。例如：OpenAI 发布了 Prism；Grok 视频变长了；NVIDIA 模型登顶了。
*   **定义点 (Definition)**：来自外部链接/知识库。例如：Prism 是一个 LaTeX 编辑器；UniRG 是医疗报告生成框架。

### 4. 知识库补充机制 (Knowledge Augmentation)

当推文提到某产品/模型/技术，但正文未提供定义，且无辅助链接时：
*   **行动**：Agent **必须**结合内置知识库（或进行必要的搜索），补充说明"它是什么"、"有什么用"。
*   **原则**：默认用户不仅想知道"发生了什么"，还想知道"这个东西为什么重要"。
*   **禁忌**：仅仅进行翻译而不做解释。例如，只说"X 发布了 Y"，而不解释 Y 是什么。

### 总结清单 (Checklist)

在生成 Markdown 摘要前，请自问：
1. [ ] 我是否包含推文中提到的具体数字（如 10s、#1、50%）？
2. [ ] 我是否查看了推文配图中的文字信息？
3. [ ] 我是否区分了"刚刚发生的动态"和"长期的技术背景"？
